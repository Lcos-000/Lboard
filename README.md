# WebSocket 通信服务(v0.2.0)

基于 `gorilla/websocket` 实现的实时通信层，为协作编辑提供双向消息推送能力。

## 🚀 核心特性

- **JWT 鉴权**：连接时通过 Query Token 验证身份，自动绑定用户
- **请求/响应匹配**：`requestId` 机制，客户端可追踪异步响应
- **房间管理**：Hub 管理连接注册/注销，支持按房间广播
- **心跳保活**：Ping/Pong 机制检测连接存活，自动清理死连接
- **并发安全**：Send Channel 缓冲写入，读写锁保护连接状态

## 🛠️ 核心组件

| 模块 | 说明 |
|------|------|
| `Connection` | WebSocket 连接封装，读/写协程，Send Channel 缓冲 |
| `Hub` | 全局连接注册表，房间映射，消息广播入口 |
| `Message` | 统一消息结构 `{type, requestId, roomId, payload}` |
| `Gateway` | HTTP Upgrade 处理，JWT Token 提取与校验 |
| `Handler` | 消息路由，`ping`/`join_room` 业务处理 |

## 📨 消息协议

### 消息结构

```json
{
  "type": "join_room",
  "requestId": "req-1",
  "roomId": "room-id",
  "payload": {}
}
```

| 字段 | 说明 |
|------|------|
| `type` | 消息类型标识 |
| `requestId` | 请求 ID，用于匹配异步响应 |
| `roomId` | 目标房间 ID |
| `payload` | 业务负载，类型由 `type` 决定 |

### 消息类型

| 消息 | 方向 | 说明 |
|------|------|------|
| `hello` | 服务端→客户端 | 连接建立确认，携带 `userId` |
| `ping` / `pong` | 双向 | 心跳检测，30s 间隔 |
| `join_room` | 客户端→服务端 | 请求加入房间 |
| `join_room_ack` | 服务端→客户端 | 加入房间成功响应 |
| `error` | 服务端→客户端 | 错误响应，含 `code` 和 `msg` |

## ⚙️ 连接生命周期

```
客户端连接 → JWT 鉴权 → Upgrade → 创建 Connection
    → Hub.Register → 发送 hello → 读写循环
    → (心跳保活) → 连接断开 → Hub.Unregister → 清理资源
```
